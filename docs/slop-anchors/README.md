# Slop anchors

Curated negative visual references used by `hooks/scripts/lib/anti-anchors.mjs` to tell the model **do not produce output that looks like these references**. This is the Sprint 8 Task 23 counterpart to the positive anchors in `taste/accepted-examples.jsonl` (Sprint 6) — instead of "match this", it's "avoid this".

Each category folder contains:

- `manifest.json` — metadata (category name, description, avoid-reasoning, avoid_families)
- 3 image files (PNG or JPEG, ≤ 300 KB each, ideally ≤ 1200 × 800) representing canonical instances of the slop category

The `anti-anchors.mjs` loader walks categories, reads each `manifest.json`, and picks a relevant subset based on the active style's avoid-family. For example:

- Generating in `fintech-trust` style → sample `saas-default-blue-hero` + `generic-feature-3up` anti-anchors
- Generating in `brutalism-feral` → sample `glassmorphism-gradient-card` + `neumorphism-pillow` (the *opposite* aesthetic) as anti-anchors

## Categories shipped

| Folder | What it captures | Primary avoid-families |
|---|---|---|
| `saas-default-blue-hero/` | The single most recognisable AI-landing-page: default Tailwind blue CTA, stock-photo hero, 3-column features | marketing, saas, b2b |
| `cyberpunk-cyan-glow/` | Cyan-on-dark with colored shadows — generic sci-fi HUD aesthetic | dashboard, saas, modern |
| `glassmorphism-gradient-card/` | Frosted-glass cards on gradient backdrop, Stripe-2022 copy | saas, marketing, dashboard |
| `neumorphism-pillow/` | Soft shadows + inset highlights on neutral surfaces, 2020 Material revival | dashboard, saas |
| `generic-feature-3up/` | Icon + heading + paragraph × 3 in grid-cols-3 — the universal features section | marketing, landing, saas |

## How to add a new category

1. Create a folder `docs/slop-anchors/<category-slug>/`
2. Add `manifest.json`:

   ```json
   {
     "category": "kebab-case-name",
     "description": "One-sentence what-this-captures.",
     "avoid_reasoning": "Why output that resembles these references is bad. Keep it mechanical — 'reads as generic 2023 SaaS' beats 'looks boring'.",
     "avoid_families": ["saas", "marketing"],
     "images": [
       { "file": "example-1.png", "source": "ai-generated via bolt.new, 2026-04-22", "caption": "Default-blue CTA + 3-column features" }
     ]
   }
   ```

3. Add 2–5 images. Sources matter — fair-use-safest options:
   - AI-generated screenshots from bolt.new / v0.dev with "build me a landing page" prompt and zero style direction
   - Your own mockups of the pattern
   - Screenshots of open-source demos that explicitly permit reuse

   **Avoid**: logos of real brands, screenshots of competitor products, anything under a restrictive licence.

4. The loader picks it up on next run — no registration step.

## Curation criteria

A reference qualifies as an anti-anchor when it demonstrably:

- **Appears frequently** across AI-generated UI output (testable: run 10 generations of a landing page with no style direction, count overlaps)
- **Is generic** — interchangeable with 40 other pages with the same role
- **Is instantly recognisable** — a designer skimming the reference thinks "I've seen this 100 times this year"

Specific-brand homages (e.g. a faithful Apple-site recreation) are NOT anti-anchors. They're style choices that might be wrong for a given brief, but they're not slop.

## Whitelist consideration

If you add an anti-anchor that happens to match a curated style (e.g. a cyberpunk anti-anchor that looks like `neon-dystopia`), add that style to the anti-anchor's `excludes_styles` list in `manifest.json`:

```json
{
  "excludes_styles": ["neon-dystopia", "y2k-futurism"]
}
```

The anti-anchor loader skips the entry when the active style is in `excludes_styles`. This prevents telling a brutalist style "don't look like brutalism".

## What's NOT in this directory

- Actual image binaries are **not auto-generated by tooling**. A maintainer adds them manually when the category is curated. Placeholder categories ship with manifest-only + a `PLACEHOLDER` file explaining what to capture.
- Copyrighted material of third parties
- User data of any kind
